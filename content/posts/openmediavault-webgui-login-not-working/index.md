OpenMediaVault - это отличная система для управления хранением файлов, которая работает по принципу "настроил и забыл".

Совсем недавно потребовалось попасть в веб-интерфейс управления такой системы, НО учетные данные для входа система принимать отказалась. Ниже возможные решения.<!--more-->

* * *

**Стандартные логин\\пароль**

WEB

- admin
- openmediavault

SSH

- root
- openmediavault

* * *

**Забыли пароль web-интерфейса**

login ssh --->

_omv-firstaid_ \---> меняем пароль

* * *

**Изменили пароль, но авторизация не происходит**

```
df -h
```

Такая ситуация возможна, если на системном разделе закончилось место. Это так? - освободите его и авторизуйтесь.

* * *

**Ищем 10 самых больших файлов в "/"**

```
find / -mount -type f -ls 2> /dev/null | sort -rnk7 | head -10 | awk '{printf "%10d MB\t%s\n",($7/1024)/1024,$NF}'
```

* * *

**Ищем 10 самых больших файлов в домашней директории**

```
find /home -mount -type f -ls 2> /dev/null | sort -rnk7 | head -10 | awk '{printf "%10d MB\t%s\n",($7/1024)/1024,$NF}'
```

* * *

**Самые тяжелые файлы в текущей директории**

```
du -sk * | sort -rn
```
