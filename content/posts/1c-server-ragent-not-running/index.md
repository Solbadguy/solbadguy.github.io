Не так давно столкнулся с проблемой на сервере 1С (Ubuntu 16.04) - аутсорсинговая компания решила поэксперементировать на проде, в результате чего на системном разделе закончилось место. <!--more-->

Казалось бы избитая проблема, которая решается удалением\\переносом некоторой части файлов куда-то еще. Однако в этом случае подобный подход не подошел. Клиенты по прежнему получали сообщение, о том, что сервер отвергает их запросы на подключение.

### Проблема:

* * *

Статус службы говорил о том, что она не работает:

```
1C:Enterprise 8.3 server status:
Init script: STARTED.
     Ragent: NOT RUNNING.
```

netstat так же подсказал, что сервер не слушает подключения на порту 1540

```
netstat -atn |grep 0.0.0.0:15

tcp 0 0 0.0.0.0:1541 0.0.0.0:* LISTEN
```

### Решение впрочем оказалось тривиальным:

* * *

**Останавливаем службу:**

```
sudo service srv1cv83 stop

```

**Удаляем папку /reg\_1541 со всем ее содержимым:**

```
rm -R /home/usr1cv8/.1cv8/1C/1cv8/reg_1541 
Либо переименуем ее
mv /home/usr1cv8/.1cv8/1C/1cv8/reg_1541 /home/usr1cv8/.1cv8/1C/1cv8/reg_1541_bk
```

После этого через остастку 1С сервера придется по новой создать кластер и завести БД.

**Запускаем службу:**

```
sudo service srv1cv83 start

1C:Enterprise 8.3 server status:
Init script: STARTED.
     Ragent: RUNNING.

```
