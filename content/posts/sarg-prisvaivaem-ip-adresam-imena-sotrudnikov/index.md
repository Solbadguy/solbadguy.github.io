Я больше чем уверен, что в большинстве организаций мало кто знает, что такое ip-адрес, соответственно воспринимать такую информацию стороннему человеку будет трудно. Давайте разберемся как в SARG назначить айпи-адресам имена сотрудников офиса.<!--more-->

Сразу к практике, через пару минут вместо холодных и непонятных для вашего руководства ip-адресов, можно будет увидеть имена того самого Лехи и Руслана.

В файле конфигурации саржа (/etc/sarg/sarg.conf) раскомментируем строку и сохраняем изменения

```
usertab /etc/sarg/usertab
```

Открываем сам юзертаб:

```
sudo nano /etc/sarg/usertab
```

Вносим в него дополнения по аналогии с примером:

192.168.0.12 Kot 192.168.0.13 Vika 192.168.0.14 Ira 192.168.0.15 Valery

Для проверки работы запускаем скрипт создания отчета еще раз, в новом списке вместо айпи адресов будут красоваться указанные вами имена.
